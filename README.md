# sweetart-bakery-web

# Опис проєкту
Цей проєкт являє собою веб-сайт для кондитерської SweetArt Bakery.  
Він включає основні сторінки: головну, каталог продукції, контакти та сторінку замовлення.  
Користувачі можуть переглядати різні категорії тортів і тістечок, а також оформлювати замовлення через форму.

# Опис Технологій та Функціональності

### 1. HTML: Структура та Семантика
HTML у проєкті виконує роль фундаменту, забезпечуючи логічну та семантичну структуру вмісту. Використовуються сучасні елементи HTML5 (`<header>`, `<nav>`, `<main>`, `<section>`, `<footer>`) для забезпечення чистої структури, що покращує доступність та індексацію. Навігаційне меню є єдиним на всіх сторінках, а активна сторінка позначається класом `active`. Форма замовлення (`orders.html`) має усі необхідні поля та унікальні `id` для роботи з JavaScript-валідацією.

### 2. CSS: Дизайн та Організація Макету
CSS відповідає за візуальне оформлення та адаптивність сайту. Стилі організовані модульно (`base.css`, `layout.css`, `components.css`) та використовують Google Fonts.
Для складних секцій (картки, навігація) використовується **Flexbox**. Сітка продуктового каталогу на `catalog.html` реалізована за допомогою **CSS Grid** (`repeat(auto-fit, minmax(...))`), забезпечуючи відмінну адаптивність. Макет є *responsive* та має медіа-запити для коректного відображення на різних пристроях. У `components.css` визначено класи **`body.dark-theme`** для повної зміни колірної схеми, а також стилі для відображення помилок валідації (`.error-message`, червоні бордери).

### 3. JavaScript-функціональність (Лабораторна робота №4)
Інтерактивність реалізована у файлі `js/main.js` за допомогою **DOM, Events, Validation** та **LocalStorage**.
Реалізовано кастомну валідацію форми замовлення, яка спрацьовує при спробі відправки (`submit`). Поля "Ім'я", "Email" та "Повідомлення" перевіряються на відповідність правилам. Поля "Ім'я", "Email" та "Повідомлення" мають обробники події `blur`, які миттєво виділяють поле червоним, якщо введене значення є некоректним (Live Validation). Реалізовано кнопку перемикання теми, яка змінює клас `dark-theme` на `<body>`. Вибір теми зберігається у **LocalStorage** для збереження стану між сесіями. Додано відображення поточної дати у футері, динамічна підсвітка назв продуктів у каталозі та механізм акордеону ("Показати більше") на головній сторінці.
Додано керування масштабом шрифту на сторінці за допомогою клавіш **`ArrowUp`** та **`ArrowDown`**.
